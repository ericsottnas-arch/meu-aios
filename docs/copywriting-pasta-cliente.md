## ⚡ WORKFLOW OBRIGATÓRIO

### Antes de Escrever QUALQUER Copy

1️⃣ **Ler Tom de Voz**
```
Abra: 🖊️ Copywriting/📋 Tom de Voz e Estilo - [CLIENTE].txt

Procure por:
  • Jargões preferidos
  • Frases características
  • Estrutura argumentativa
  • Tom conversacional
  • Emotional triggers
  • Palavras proibidas
```

2️⃣ **Consultar Copy Anterior**
```
Revisite headlines/emails/posts antigos:
  • Qual copy performou melhor?
  • Qual tom gerou mais engajamento?
  • Que elementos repetem?
  • O que mudou recentemente?
```

3️⃣ **Revisar Público-Alvo**
```
Consulte profile.md ou ANALISE-ATLAS.md:
  • Quem é o público?
  • Quais são os pain points?
  • Qual é o ICP?
  • Qual é o buyer journey?
```

### Depois de Escrever Copy

4️⃣ **Salvar na Pasta Correta**
```
Escolha subpasta apropriada:
  • Headlines → 📂 Headlines/
  • Email → 📂 Emails/
  • Instagram/LinkedIn → 📂 Social/
  • Landing page → 📂 Landing Pages/
  • Google/Facebook Ads → 📂 Anúncios/
  • Sales call script → 📂 Scripts/
```

5️⃣ **Nomear com Padrão**
```
Padrão: YYYYMMDD_tipo_descricao.md

Exemplos:
  • 20260225_headline_novo_produto.md
  • 20260225_email_sequence_frio.md
  • 20260225_instagram_post_story.md
  • 20260225_sales_page_beneficios.md
```

6️⃣ **Documentar Performance**
```
Adicione ao início do arquivo:

---
**Data:** 25/02/2026
**Tipo:** Headline
**Status:** Em teste
**CTR Esperado:** 3-5%
**Público:** Advogados
**Framework:** AIDA
---

[Copy aqui]
```

## 💡 Melhores Práticas

### ✅ FAÇA:

- ✅ Ler Tom de Voz ANTES de escrever
- ✅ Consultar copy anterior para consistência
- ✅ Documentar framework utilizado
- ✅ Registrar data e tipo de copy
- ✅ Organizar em subpastas corretas
- ✅ Nomear arquivos com padrão
- ✅ Atualizar Tom de Voz conforme aprende sobre cliente
- ✅ Documentar A/B tests realizados
- ✅ Arquivar copy desatualizado

### ❌ NÃO FAÇA:

- ❌ Deixar copy em raiz ou pasta errada
- ❌ Ignorar Tom de Voz estabelecido
- ❌ Escrever sem consultar copy anterior
- ❌ Nomes genéricos de arquivo (copy1, final, versão2)
- ❌ Misturar tipos de copy em 1 arquivo
- ❌ Deixar copy sem documentação
- ❌ Mudar tom sem avisar equipe
- ❌ Deletar copy antigo sem arquivar
