# Regras de Conteudo para Redes Sociais (@nova)

> Consultar ao produzir conteudo para Instagram, LinkedIn, TikTok, qualquer rede social.
> Ver tambem: [[universal]], [[copy-escrita]]

---

## [CRITICAL] Dados reais — NUNCA inventar

- NUNCA criar casos ilustrativos/ficticios sem deixar claro que sao ficticios
- TODO case citado deve ser REAL, de um cliente Syra Digital, com dados verificaveis
- Se nao tem case real para aquele tema → nao usa case
- TODO dado e estatistica DEVE ter fonte especificada

**Funil completo obrigatorio:**
Ao citar case com numeros, incluir SEMPRE:
investimento → leads → agendamentos → vendas → faturamento
(nunca parar no meio do funil)

---

## [CRITICAL] PROIBIDO: travessao (—)

Ver [[universal]] — vale para TODO conteudo de redes sociais tambem.
Substituir por: ponto final, virgula, hifen simples (-) ou reestruturar a frase.

---

## [HIGH] Roteiros de video — tom e formato

- Tom: natural, conversacional. Testar: "Eric falaria isso assim numa conversa real?"
- Se parece "engessado" ou "de manual" → reescrever
- Texto direto, SEM aspas ao redor do roteiro
- Cada ideia = paragrafo separado. Max 3-4 linhas por paragrafo

---

## [HIGH] Tipos de formato por documento

- Documento de batch para gravacao: conter APENAS roteiros de video (F5 Reel)
- Conteudo estatico (F1 Frase, F2 Carrossel, F3 Thread, F4 Tweet) vai em documento SEPARADO
- Nunca misturar conteudo de gravacao com conteudo estatico no mesmo doc

---

## [HIGH] CTA — Call to Action palpavel

CTAs devem oferecer algo CONCRETO e PALPAVEL:
- "checklist com X itens"
- "sequencia de X mensagens"
- "planilha de X numeros"

NUNCA usar: "diagnostico", "analise", "consultoria" → palavras vagas que nao convertem

---

## [HIGH] Roteiros com entrevistas

- Oferecer 4-5 variacoes de perguntas para cada momento de entrevista
- Videomaker precisa de flexibilidade — nao dar apenas uma opcao
- Fala de autoridade deve soar como conversa real, nao copy formatada

---

## [HIGH] Fontes e fontes de dados

Ao usar dados, estatisticas, tendencias:
1. Fontes internas (transcrinos de calls, dados Syra) sao PREFERIDAS sobre pesquisa externa
2. Consultar `docs/clientes/*/knowledge-base/` e transcricoes ANTES de ir para web
3. Quando usar pesquisa externa, citar a fonte no documento

---

## [MEDIUM] Quebrar texto nos roteiros

- Blocos de texto muito juntos dificultam leitura
- Cada mudanca de argumento, exemplo ou instrucao = novo paragrafo
- Max 3-4 linhas por bloco
