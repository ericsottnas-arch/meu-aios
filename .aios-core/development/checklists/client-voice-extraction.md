# Client Voice Extraction Checklist

**Quando:** SEMPRE antes de escrever copywriting para um cliente
**Responsável:** Copy-Chef (e especialistas que seguem)
**Status:** ⚠️ OBRIGATÓRIO

---

## ✅ CHECKLIST: Tom de Voz & Estilo do Cliente

### 1. PROCURAR DOCUMENTAÇÃO EXISTENTE
- [ ] Arquivo "📋 Tom de Voz e Estilo - [CLIENTE].txt" no `/docs/clientes/`?
- [ ] Arquivo no Drive do cliente (Syra Digital/1. Clientes/[CLIENTE]/)?
- [ ] Briefing ou análise anterior do cliente?
- [ ] Exemplos de copy anterior que funcionou bem?

### 2. SE NÃO EXISTIR, EXTRAIR DO ZERO

#### A. JARGÕES & TERMINOLOGIA
- [ ] Palavras-chave que o cliente usa FREQUENTEMENTE
- [ ] Termos técnicos preferidos vs. evitados
- [ ] Expressões únicas do cliente
- [ ] Gírias/regionalismos
- **Exemplo:** Dr. Erico usa "segurança jurídica", "estruturar", "retaguarda"

#### B. FRASES CARACTERÍSTICAS
- [ ] Aberturas típicas ("Olha...", "Deixa que...", "A questão é...")
- [ ] Phrases de desenvolvimento
- [ ] Fechamentos/transições preferidas
- [ ] Exclamações/ênfase ("Gratidão gente!", "Deixa que...")
- **Exemplo:** Erico: "Deixa eu descomplicar isso...", "Muitos profissionais têm dúvida sobre isso"

#### C. ESTRUTURA ARGUMENTATIVA
- [ ] Como o cliente apresenta problemas?
- [ ] Como o cliente estrutura soluções?
- [ ] A sequência lógica é: Problema → Lei → Solução → Resultado?
- [ ] O cliente começa com pergunta, história ou fato?
- **Exemplo:** Erico: Validação → Legislação → Implicações → Recomendação → Tranquilidade

#### D. TOM CONVERSACIONAL
- [ ] Formal ou casual?
- [ ] Humanizado ou corporativo?
- [ ] Consultivo (pergunta) ou assertivo (afirma)?
- [ ] Autoridade + acessibilidade?
- [ ] Usa emojis?
- **Exemplo:** Erico = Formal+Humanizado+Consultivo

#### E. GATILHOS EMOCIONAIS USADOS
- [ ] Dor/Medo que o cliente comunica?
- [ ] Aspiração/Ganho que promete?
- [ ] Tranquilidade/Segurança que oferece?
- [ ] Social proof que usa?
- **Exemplo:** Erico: Dor=Insegurança, Ganho=Tranquilidade, Proof=Depoimentos reais

#### F. O QUE O CLIENTE NÃO FAZ
- [ ] Evita juridiquês?
- [ ] Não é agressivo?
- [ ] Não cria pressão artificial?
- [ ] Nunca faz X, Y ou Z?
- **Exemplo:** Erico não usa jargão, não é ameaçador, não é impessoal

### 3. DOCUMENTAR TUDO
- [ ] Criar arquivo: "📋 Tom de Voz e Estilo - [CLIENTE].txt"
- [ ] Salvar no `/docs/clientes/` E no Drive do cliente
- [ ] Estruturar com seções (Características, Jargões, Frases, Estrutura, Tom, Gatilhos, Não-Faz)
- [ ] Adicionar EXEMPLOS reais extraídos do cliente

### 4. USAR NA CÓPIA
- [ ] ✅ Referenciar arquivo ao iniciar projeto de copy
- [ ] ✅ Aplicar tom + jargões em CADA peça escrita
- [ ] ✅ Validar que copy respeta padrão de argumentação
- [ ] ✅ Evitar TUDO que está na lista "Não-Faz"
- [ ] ✅ Se cliente mudar tom: atualizar arquivo imediatamente

### 5. MANTER ATUALIZADO
- [ ] Revisar todo trimestre se cliente é recorrente
- [ ] Após cada projeto: feedback do cliente altera tom?
- [ ] Performance de copy: que tom funcionou melhor?
- [ ] Atualizar arquivo se novas descobertas
